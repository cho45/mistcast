# 計画：ベースバンド・リサンプリング・アーキテクチャへの移行

## 1. 実装の目的と仮説
- **目的**: 入力サンプルレート（44.1kHz / 48kHz等）に関わらず、DSSS内部ロジックを定数 `INTERNAL_SPC` で定義された固定整数レートで動作させ、タイミングドリフト（累積誤差）を排除する。
- **仮説**: 内部処理レートをチップレートの整数倍（初期値として奇数の $spc=3$ を想定）に固定することで、チップ中央のサンプリング精度を向上させ、かつ任意の入力レートに対して一貫した同期・復調性能を確保できる可能性がある。

## 2. 実装タスクリスト

### フェーズ1：基盤・共通定義の整備
- [ ] **[Task 1.1] `DspConfig` の拡張**: 
    - `INTERNAL_SPC` 定数を導入（初期値 3）。
    - 内部処理用（`INTERNAL_SPC`固定）の構成を生成するメソッドを追加し、入力レートと処理レートを定義上で分離する。
- [ ] **[Task 1.2] ユーティリティの整理**: `sample_at_fractional` などの補間関数を `common` モジュールへ整理し、リサンプラや他モジュールと共有可能にする。

### フェーズ2：`Decoder` 構造のリファクタリング
- [ ] **[Task 2.1] メンバ変数の刷新**: `DecimatingRrcFilter` を廃止。`Resampler` (I/Q 独立2系統) と標準 `RrcFilter` を導入する。
- [ ] **[Task 2.2] 初期化ロジックの実装**: 入力レートから $f_{s\_proc} = R_c \times INTERNAL\_SPC$ を算出し、RRC信号帯域を保護しつつエイリアシングを防ぐ適切な `cutoff_hz` で `Resampler` を構成する。

### フェーズ3：処理パイプラインの再実装 (`process_samples`)
- [ ] **[Task 3.1] 混合処理（高レート）**: 入力レートのまま複素混合を行い、I/Q信号を生成する。
- [ ] **[Task 3.2] リサンプリング（ベースバンド）**: 混合後の I/Q 信号それぞれに対し、`Resampler` を適用して内部処理レート（`INTERNAL_SPC`相当）へダウンサンプリングする。
- [ ] **[Task 3.3] マッチドフィルタリング（低レート）**: 内部処理レートにおいて `RrcFilter` を適用し、信号を抽出する。
- [ ] **[Task 3.4] 内部ロジックの接続**: `INTERNAL_SPC` 固定の世界として、`SyncDetector` およびトラッキングロジックへバッファを供給する。

### フェーズ4：検証と事実確認（品質保証）
- [ ] **[Task 4.1] インパルス応答整合性テスト**: 縦続フィルタ（Tx RRC -> Resample -> Rx RRC）がナイキスト基準を満たしているかを数学的に検証する（ISIの計測）。
- [ ] **[Task 4.2] 44.1kHz 再現検証**: `dsss_e2e_eval` を用い、44.1kHz 環境下で BER 0 が達成される事実を確認する。
- [ ] **[Task 4.3] パフォーマンス計測**: WASM環境を想定し、計算負荷増大がリアルタイム処理のデッドライン（通常 10-20ms 程度）に与える影響を計測する。

### フェーズ5：`Modulator` のリファクタリング（送信側対称化）
- [ ] **[Task 5.1] 送信側リサンプラの導入**: `Modulator` に `resampler_i/q` を導入。内部処理用レート ($spc=INTERNAL\_SPC$) で波形を生成し、出力レートへ変換する構成に刷新する。
- [ ] **[Task 5.2] 送信パイプラインの刷新**: 「チップ生成 $\to$ RRC (低レート) $\to$ Resample (変換) $\to$ Mix (高レート)」の順序に組み替え、物理的なチップレートの正確性を確保する。
- [ ] **[Task 5.3] 44.1kHz E2E 完全検証**: 対称化された Modulator と Decoder を用い、44.1kHz 設定での通信が成功（BER 0）することを客観的に確認する。

## 3. 残存する不確実性と技術的リスク
- **境界条件の不連続性**: `Resampler` が内部に保持する `history` バッファが、ストリーム処理の境界において位相やタイミングの微小な不連続を引き起こし、同期スコアを低下させる可能性がある。
- **フィルタ特性の干渉（Double-Filtering）**: `Resampler` の LPF 特性と `RrcFilter` の特性が重なることで、通過帯域の端（ロールオフ部分）が意図せず減衰し、符号間干渉（ISI）が悪化する懸念がある。
- **計算リソースの不足**: フィルタ段数が増える（Mix $	o$ 2xResample $	o$ 2xRRC）ことにより、低スペックなモバイル端末において WASM 実行速度が追いつかないリスクがある。

## 4. 完了判定基準
1. 44.1kHz および 48kHz の双方のシミュレーション環境において、$\sigma=0.3$ 条件下で BER=0 かつ `p_complete=1.0` が達成されること。
2. ソースコードから `choose_decimation_factor` や「サンプルレートがチップレートの倍数であること」を前提とした条件分岐が完全に排除されていること。
