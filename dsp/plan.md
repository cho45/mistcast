# 実装計画 (Implementation Plan) : M-Ary DQPSK 準コヒーレントHMMの厳密化

## 1. 目的と背景 (Why)
現在の M-Ary パケット復調処理における HMM (Hidden Markov Model) は、差動変調 (DQPSK) の数学的性質（エッジ依存性）を無視した「ノード評価」を行っており、かつパイロット信号も差動で挿入されているため、絶対位相のアンカーとして機能していません。
これを「絶対位相パイロット」と「絶対位相空間でのHMM平滑化 → 差動LLR抽出」という正しい数学的モデルに再構築し、ノイズ耐性とサイクルスリップ耐性を極限まで高めます。

## 2. 理論的アプローチ (Mathematical Model)
1. **絶対位相 HMM の構築**:
   隠れ状態 $\theta_m$ を「時刻 $t$ におけるチャネルの絶対位相」と定義します（$M=32$ 状態の離散グリッド）。
   観測値 $y_t$ (逆拡散後の複素振幅) に対する尤度 $P(y_t | \theta_m, h_k)$ を各 Walsh 仮説 $h_k$ と共に計算し、Forward-Backward アルゴリズムで各時刻・各状態の周辺化事後確率 $\gamma_t(m) = P(\theta_{t,m} | y_{1:T})$ を求めます。
2. **絶対位相パイロットの制約**:
   時刻 $t$ がパイロットスロットの場合、送信される Walsh 系列 ($h_0$) と絶対位相（例：$\phi_0 = 0$）は既知です。HMM の観測尤度をこの既知情報に固定することで、強力な絶対位相アンカーとして機能させます。
3. **事後確率空間での差動 LLR 抽出 (最重要)**:
   時刻 $t$ の DQPSK シンボル $s_t$（4つの位相差仮説 $d \in \{0, \pi/2, \pi, -\pi/2\}$）の事後確率は、平滑化された絶対位相の同時事後確率 $P(\theta_t, \theta_{t-1} | y_{1:T})$ のエッジ（枝）の周辺化として計算します。
   計算量を下げるため、Forward ($\alpha$) と Backward ($\beta$) を使った効率的な状態遷移計算を利用して LLR を導出します。

## 3. 実装ステップ (Task List)

### Step 1: エンコーダの改修 (modulator.rs)
- **対象**: `src/mary/modulator.rs` の `encode_frame` またはパケット生成ループ。
- **内容**: パイロットスロット挿入時、直前の `prev_phase` に `delta=0` を足す（差動）のではなく、**強制的に特定の絶対位相（例：`0` = +1）を送信**するように仕様変更します。
  - この際、パイロット送信後の次のデータシンボルが、**パイロットの絶対位相を基準とした差動**になるように `prev_phase` を更新するロジックを担保します。

### Step 2: HMM 観測尤度計算の再構築 (packet_decoder.rs)
- **対象**: `joint_symbol_logs_for_phase_state` および `dqpsk_phase_log_metrics`。
- **内容**: 
  - トラッキングループが推定した `prev_phase` に依存したノード計算を完全に廃止します。
  - 状態 $\theta_m$ における「絶対位相の観測尤度」を純粋に $y_t \cdot e^{-j\theta_m}$ と各仮説シンボルの距離（あるいはベッセル関数近似）で再定義します。
  - パイロットスロット時は、既知の絶対位相に対する尤度のみを計算し、他の状態の尤度を `-INF` に落としてアンカーとします。

### Step 3: Forward-Backward アルゴリズムのクリーンアップ (packet_decoder.rs)
- **対象**: `symbol_llrs_with_phase_hmm`。
- **内容**: 
  - 絶対位相空間で $\alpha_t$ (Forward) と $\beta_t$ (Backward) を計算する処理を、数学的に透明で副作用のない形に整理します。

### Step 4: 差動 LLR (DQPSK) の抽出ロジック実装 (packet_decoder.rs)
- **対象**: `symbol_llrs_from_joint_post` を置き換える新関数。
- **内容**:
  - $\alpha_{t-1}(m_{prev})$ と $\beta_t(m_{curr})$ と観測尤度、および遷移確率を用いて、「位相差が $d$ になるような状態ペア $(m_{prev}, m_{curr})$」の結合確率を周辺化し、2bit の LLR（bit0, bit1）を生成します。

### Step 5: テストによる理論的正しさの数学的証明 (packet_decoder.rs)
- **内容**: 
  - パイロットスロットが HMM の $\alpha, \beta$ を正しく制約（ピン留め）していることを確認するテスト。
  - 差動 LLR が、スリップ（絶対位相の90度回転）が発生してもデータを正しく復元できる（差動の性質が維持されている）ことを証明するテスト。

## 4. リスクと対策
- **互換性**: パイロットのエンコード規則が変わるため、変更後は過去の生波形（IQデータ）との互換性が失われます。これはアーキテクチャの根幹修正であるため不可避ですが、シミュレータ/テスト内で生成と復号が完結するためシステム全体としての安全性は保たれます。
