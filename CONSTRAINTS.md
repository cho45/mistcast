# 音響DSSS通信：設計制約とパラメータ選定基準

室内環境におけるスマートフォン等のデバイス間通信を想定し、FSKを凌駕するロバスト性と、ファウンテンコードを活かした実用的なデータレートを両立するための制約条件を整理する。

## 1. 物理的制約（Physical Constraints）

### 1.1 帯域幅の限界
*   **サンプリングレート**: 44.1kHz / 48kHz。
*   **ナイキスト周波数**: 22.05kHz / 24kHz。
*   **デバイス特性**: 一般的なスマートフォンのスピーカーおよびマイクは 18kHz〜20kHz 以上で急激に感度が低下する。
*   **制約**: 信号の占有帯域上限（$f_c + R_c(1+\alpha)/2$）は **20,000 Hz** を超えてはならない。

### 1.3 音響的隠蔽性（Acoustic Concealment）
*   **DSSSの利点**: 信号を広帯域に拡散することで電力密度を下げ、背景ノイズに紛れ込ませることができる。
*   **不快感の排除**: 特定の周波数が目立つ「ピー」というトーン音を避け、ユーザーにとって不快でない（あるいは気づかない）「ザー」という白色雑音に近い質感を目指す。
*   **制約**: 
    *   データの偏りによるトーン発生を防ぐため、物理層でのスクランブルを徹底する。
    *   特定の周波数ピークを作らず、広い帯域に分散させる。
    *   下限周波数は聴覚上不快でなければ 50Hz 程度まで下げても良い。

## 2. プロトコル制約（Protocol Constraints）

### 2.1 同期頻度と成功率
*   **設計思想**: 「同期頻度を上げることが、実環境での通信成功率に最も寄与する」。
*   **ファウンテンコードとの親和性**: 1つの長いバーストを送るよりも、短いバーストを大量に投げるほうが、時間的なフェージングや突発的ノイズに対して生存率が高まる。
*   **制約**: **1 Sync Burst あたりのパケット数を最小化（1〜2パケット）**し、常に最新のタイミング・位相・AGC状態でデコードを開始できるようにする。

### 2.2 フレーム長とCFO（周波数偏差）
*   **特性**: 送受信機間の個体差による数Hz〜数十HzのCFOが発生する。
*   **影響**: 1シンボルの時間（$SF/R_c$）が長いほど、バースト内での累積位相回転が大きくなる。
*   **制約**: 同期頻度を上げる（フレームを短くする）ことで、CFOによる累積誤差が判定を誤らせる（例：45度以上の回転）前にフレームを完結させる。

### 2.3 DSSSのアイデンティティ（対FSK）
*   **目標**: FSKに対し、ロバスト性とスペクトル効率の双方で勝る必要がある。
*   **処理利得（Processing Gain）**: $SF$ が低いと単なる低速BPSKになり、DSSSのメリット（低SNR耐性）が失われる。
*   **制約**: **$SF \ge 15$** ($11.7	ext{dB}$ 利得) を下限とする。

## 3. パラメータ間の論理的トレードオフ

| パラメータ | 増加させた際の影響 (+) | 減少させた際の影響 (-) |
| :--- | :--- | :--- |
| **チップレート $R_c$** | スループット向上、占有帯域拡大 | **マルチパス耐性向上**、CFO耐性向上（シンボル短縮） |
| **拡散率 $SF$** | **SNR耐性向上（処理利得）** | スループット向上、CFO耐性向上（シンボル短縮） |
| **ロールオフ $\alpha$** | 帯域外放射の抑制、ISI低減 | 帯域効率の向上（より高い $R_c$ が可能） |

## 4. 最適化の評価関数

以下の優先順位でパラメータを評価する：

1.  **P-Complete (1.0)**: マルチパス（10ms遅延）+ ノイズ（$\sigma=0.2$）環境下での完遂率。
2.  **Latency-to-Sync**: 最初のパケットが正しく受信されるまでの平均時間（同期頻度の指標）。
3.  **Effective Throughput**: 冗長性を除いた純粋な BPS。

---
**結論**: 高い $R_c$ で速度を稼ぐのではなく、**「低い $R_c$ と適切な $SF$、そして極めて短いフレーム長」**の組み合わせこそが、室内音響通信におけるDSSSの最適解である。
