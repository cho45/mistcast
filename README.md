# Mistcast

Mistcastは、ブラウザ上で動作する**音響データ通信（Data-over-Sound）ウェブアプリケーション**です。
スピーカーとマイクを使用し、音響空間を介したデータ伝送を実装しています。

https://github.com/user-attachments/assets/9e2b40ab-bfc8-4fa5-8e5b-f9ce8e6f0eb9

(この動画内音声の信号は現在の実装より古いため、現在のデモでデコードできません)

## 特徴

- **スピーカーとマイクによる近距離ブロードキャスト**: Wi-FiやBluetoothのような事前のペアリング設定は不要です。スピーカーから音を鳴らすだけで、周囲にある複数のデバイス（マイク）へ同時にデータを配信できます。
- **ストリームのどこからでも受信可能**: ファウンテンコード（消失訂正符号）を採用しており、送信側はデータを継続的に射出し続けます。受信側は通信の途中から参加した場合でも、必要な量のデータを受信した時点で即座に元のデータを復元可能です。
- **ブラウザの標準機能のみ**: Web Audio API および WebAssembly (SIMD) を活用し、専用のアプリやプラグインをインストールすることなく、Webブラウザを開くだけで動作します。

## 技術仕様

### 1. 物理レイヤー (物理変調)

Mistcast は、複数の物理層実装を含んでいます。

| 特徴 | DSSS (Slow) | M-ARY (Fast) |
| :--- | :--- | :--- |
| **変調方式** | DSSS + DQPSK | 16-ary Orthogonal + DQPSK |
| **伝送効率** | 2 bits / symbol | **6 bits / symbol** |
| **インターリーバ** | 12×29=348 bits | **12×29=348 bits (効率100%)** |
| **拡散率 (SF)** | 15 (M系列) | 16 (Walsh-Hadamard) |
| **チップレート** | 8,000 cps | 8,000 cps |
| **キャリア周波数** | 15,000 Hz | 15,000 Hz |
| **実効速度 (Goodput)** | **~ 352 bps** (1 pkt/burst) | **~ 1114 bps** (3 pkts/burst) |
| **マルチパス対策** | N/A | **FDE + 自動パス選択** |

- **共通基盤**:
    - **対称リサンプリング**: 入出力サンプリングレートを内部処理レート（$3 \times R_c$）へ変換し、タイミング累積誤差を抑制。
    - **RRC (Root-Raised Cosine) フィルタ**: ロールオフ係数 0.30 による波形整形。
    - **軟判定復調**: Max-Log-MAP 近似による LLR (対数尤度比) 算出と Viterbi 復号の統合。

## フレーム構造とパケット構造

Mistcastは、物理層（変調フレーム）とデータリンク層（LT符号パケット）の2層構造でデータを伝送します。

### 変調フレーム (Physical Layer Frame)
1回の同期（プリアンブル）で送信される単位です。

```text
|<--- Preamble --->|<- Sync ->|<---------- Data Symbols (Payload) ---------->|
+------------------+----------+----------------------------------------------+
| PN Sequence x 2  | 8-16 bit | DQPSK Symbols (Packet 1, [Packet 2, ...])    |
| (M-Seq or ZC)    | (DBPSK)  | (DSSS or M-ARY Spreading)                    |
+------------------+----------+----------------------------------------------+
^                  ^          ^
|                  |          +-- Payload開始 (基準位相)
|                  +-- フレーム境界確定
+-- タイミング同期 / AGC
```

- **Preamble系列**: DSSS モードは **M系列** (SF=15)、M-ARY モードは **Zadoff-Chu系列** (SF=71) を使用。
- **Sync Word長**: DSSS モードは 16ビット、M-ARY モードは 8ビットを使用。

### パケットフォーマット (Link Layer Packet)
1つのパケットは 21 バイトで構成され、消失訂正符号（ファウンテンコード）の最小単位となります。

```text
|<--- Header (3B) --->|<--- Payload (16B) --->|<-- CRC (2B) -->|
+----------+----------+-----------------------+----------------+
|  lt_k-1  |  lt_seq  |      Data Bytes       |    CRC-16      |
|  (8-bit) | (16-bit) |      (16 bytes)       |   (CCITT)      |
+----------+----------+-----------------------+----------------+
```

- **lt_k**: データを分割した際の総ソースブロック数 (1〜255)。
- **lt_seq**: ファウンテンコードのシーケンス番号。
- **データサイズ制限**: `lt_k` が 8ビット（最大255）であるため、一度に送信可能な最大データサイズは **$255 \times 16 = 4,080$ バイト**に制限されます。

### パケットの物理層へのマッピング

DSSSモードとM-ARYモードで、パケットの物理シンボルへのマッピングが異なります。

#### DSSS (Slow) モード
```
パケット (21B) → FECエンコード (174→348 bits) → インターリーバ (348 bits, paddingなし)
                                     ↓
                    DQPSK変調 (2 bits/symbol) → 174 symbols
                    ↓
                    DSSS拡散 (SF=15) → 2,610 chips
```

#### M-ARY (Fast) モード
```
パケット (21B) → FECエンコード (174→348 bits) → インターリーバ (348 bits, paddingなし)
                                     ↓
                    MaryDQPSK変調 (6 bits/symbol) → 58 data symbols
                    ↓
                    Walsh拡散 (SF=16) → 928 chips
```

**主な違い**:
- **インターリーバパディング**: DSSS/M-ARY ともに 12x29=348 bits に揃えているため、追加パディングは不要です。
- **変調効率**: DSSSは2 bits/symbol、M-ARYは6 bits/symbol（3倍の密度）
- **拡散系列**: DSSSはM系列(SF=15)で直接拡散、M-ARYはWalsh系列(SF=16)による直交変調。

## 同期捕捉 (Synchronization)

安定した同期を実現するため、フレームのセクションごとに変調を切り替えます。

### プリアンブル（Preamble）
タイミング同期とキャリア同期に使用されます。

| モード | 系列 | 拡散率 | パターン | 特徴 |
| :--- | :--- | :--- | :--- | :--- |
| **DSSS (Slow)** | M系列 | SF=15 | [M, -M] | ノンコヒーレント相関による軽量な自己相関 |
| **M-ARY (Fast)** | Zadoff-Chu | SF=71 | [ZC, -ZC] | 自己相関サイドローブがゼロ、FDEのためのCIR推定に利用 |

- **最終シンボル反転**: 両モードとも最後のシンボルを反転させ、同期の位相曖昧さを排除
- **繰り返し回数**: `preamble_repeat` パラメータで調整可能（デフォルト2回）

### Sync Word
フレーム境界の確定とリファレンス位相の確立に使用されます。

| モード | Sync Word長 | 変調方式 | 拡散系列 |
| :--- | :--- | :--- | :--- |
| **DSSS (Slow)** | 16ビット | DBPSK | M系列 (SF=15) |
| **M-ARY (Fast)** | 8ビット | DBPSK | Walsh[0] (SF=16) |

- **固定パターン**: `0xDEAD_BEEF` (下位ビットから使用)
- **機能**: フレーム境界の確定とリファレンス位相の確立

### Handoff
- Sync Word 終了時の位相を Payload 領域の初期位相として引き継ぎ

### 3. 誤り訂正と消失訂正

- **FEC**: 畳み込み符号（**拘束長K=7**, 符号化率**R=1/2**, NASA Voyager標準）とソフト判定 Viterbi 復号。
  - 生成多項式: G1=0o171 (121), G2=0o133 (91)
  - 状態数: 2^(K-1) = 64状態
- **CRC-16**: パケット単位の誤り検出（CRC-CCITT）。
- **インターリーバ**: バースト誤り分散のためのブロックインターリーバ。
- **RLNC (Random Linear Network Coding)**: GF(256) 上の消失訂正符号(ファウンテンコード)。パケットの欠落や順序入れ替わりに強い耐性を持ちます。

### 4. M-ARYモードの高度機能

- **FDE（周波数領域等化器）**: マルチパスフェージング環境での信号品質を改善します。
  - **MMSE等化**: 最小平均二乗誤差基準による等化
  - **FFTベース処理**: オーバーラップ加算法で効率的な処理
  - **CIR推定**: プリアンブル(Zadoff-Chu)からチャネルインパルス応答を推定
- **FDE自動パス選択**: チャネル状態に応じてFDE出力と生信号のMSEを比較し、良好な方を自動選択します。
- **チャネル品質推定**: 受信信号の品質(SNR, CFO, ノイズ分散)をリアルタイムで推定します。

## 同期頻度と実効速度

Mistcastは、**「頻繁に同期（プリアンブル）を送り直す」**設計を採用しています。デフォルトでは、約 370ms ごとに同期信号が挿入されます。

### 頻繁な同期のメリット・デメリット

| メリット | デメリット |
| :--- | :--- |
| **即時復帰**: ノイズや遮蔽で同期が外れても、次のバースト（数百ms後）で即座に復帰可能。 | **オーバーヘッド**: 同期信号（33.75ms）が占める時間の分、純粋な伝送速度が低下する。 |
| **ファウンテンコードとの相性**: どの位置から受信しても、最初の同期さえ掴めれば即座にデータ断片を得られる。 | **音響的なノイズ**: 同期信号（プリアンブル）が頻繁に鳴るため、聴感上の「通信音」が目立つ。 |
| **クロックドリフト耐性**: 送受信間のサンプリングレート微差によるタイミングズレが蓄積する前にリセットされる。 | |

### モード別の伝送効率（同期周期を一定にした場合）

<img src="docs/plot_comparison.png">

同期の堅牢性（やり直しが効く頻度）を維持するため、同期周期を **約 370ms** に揃えた場合の比較です。

| モード | バースト構成 | 同期周期 | 伝送量 / バースト | 同期比率 |
| :--- | :--- | :--- | :--- | :--- |
| **DSSS (Slow)** | 1 パケット | **363.75 ms** | 128 bits | 9.3% |
| **M-ARY (Fast)** | 3 パケット | **387.75 ms** | **384 bits** |  **8.7%** |

M-ARY (Fast) モードでは、1シンボルあたりの情報密度が 3倍（6 bits/symbol）であるため、**「同期のやり直しチャンス」を減らすことなく、データ伝送量だけを約3倍に増やす**ことが可能です。また、同期信号が占める時間的割合（オーバーヘッド）も M-ARY (Fast) の方が僅かに低くなり、より効率的な通信を実現しています。

### 変調方式のトレードオフ

Mistcastでは、計算負荷と通信性能（速度・堅牢性）のトレードオフが2つのモード間で明確に分かれています。

| 特性 | DSSS (Slow) | M-ARY (Fast) | 理由 |
| :--- | :--- | :--- | :--- |
| **ノイズ・自己干渉耐性** | 低い | **高い** | DSSSは全データが位相(DQPSK)に依存するため、帯域制限による自己干渉(ISI)の位相ジッタに弱く、ベースラインエラーが生じます。M-ARYは情報の大半(4/6bits)を「位相に依存しないエネルギー判定(Walsh系列の選択)」で復号し、多値直交変調による強力な符号利得を得るため圧倒的に堅牢です。 |
| **マルチパス耐性** | 低い | 中高 | DSSSは時間領域のシンプルな追従のみ行うため遅延波による波形歪みに弱いです。M-ARYはFDE（周波数領域等化器）を搭載していますが、プリアンブル長(ZC系列長)の制約上、補償できる遅延スプレッドには限界があります。 |
| **計算負荷 (CPU)** | **低い** | 高い | M-ARYは受信時に16個の相関器の並列稼働やFFTなど、リッチな信号処理を行うためCPU負荷が高くなります。DSSSは1つの相関器のみで動作するため軽量です。 |
| **パラメータの柔軟性** | **高い** | 低い | M-ARYは直交変調の性質上、「拡散率(SF)」と「並列相関器の数(M)」が数学的に一致(SF=16, 16-ary)しなければならず、レート調整が困難です。DSSSはM系列の長さ(SF)を自由に変更しやすく、感度と速度のトレードオフを容易に設計できます。 |
| **総合評価** | 省電力・軽量 | **主力・高性能** | DSSSは計算リソースを極限まで削った軽量・柔軟な実装です。M-ARYは計算リソースを潤沢に投入し、通信理論上の直交変調の強みを引き出すことで、**「高いスループット」と「高いノイズ耐性」を両立**させています。 |

## 技術スタック

- **DSPコア (Rust / WebAssembly)**
  - 信号処理（変復調・誤り訂正など）の本体は Rust で記述され、WebAssembly にコンパイルして実行されます。
  - 環境の対応状況に応じて最適なパフォーマンスを発揮するため、ビルド時に標準版 (base) と SIMD 対応版の2つの WASM バイナリを生成し、実行時にサポート状況を判定して動的にロードし分けます。
- **Web フロントエンド (Vue 3 / Vite / TypeScript)**
  - ユーザーインターフェースおよび Web Audio API、Web Worker の連携は Vue 3 をベースに構築されています。
  - 開発およびビルドツールには Vite を採用しています。

## ディレクトリ構造

```text
.
├── src/                # Webフロントエンド (Vue 3 / TypeScript)
│   ├── components/     # UIコンポーネント (Sender/Receiver)
│   ├── worker.ts       # DSP処理を実行する Web Worker
│   └── audio-processors.ts # Web Audio API と WASM のブリッジ
├── dsp/                # DSPコアエンジン (Rust)
│   ├── src/
│   │   ├── lib.rs              # WASMパブリックAPI
│   │   ├── common/             # 数学的プリミティブ (Walsh, M系列, RRC, NCO, Resample)
│   │   ├── dsss/               # DSSS (Slow) モード: DSSS+DQPSK 実装
│   │   ├── mary/               # M-ARY (Fast) モード: 16-ary+DQPSK 実装
│   │   ├── coding/             # 符号化層 (FEC, Fountain, Interleaver, Scrambler)
│   │   └── frame/              # データリンク層 (Packet Format)
│   └── tests/          # 通信シミュレーション・ノイズ耐性テスト
└── scripts/            # WASMビルド・評価用Pythonスクリプト
```

### クイックスタート
1. **依存関係のインストール**: `npm install`
2. **WASM コアのビルド**: `npm run build:wasm`
3. **開発サーバーの起動**: `npm run dev` (`http://localhost:5173`)

- **DSP Core ユニットテスト**: `cd dsp && cargo test --release`
