# Mistcast

Mistcastは、エアギャップ環境（ネットワークに接続されていないデバイス間）での音響通信を実現するための、堅牢で数学的に厳密なDSP（デジタル信号処理）コアライブラリです。

ノイズの多い現実の音響空間でもデータを確実に伝送するため、スペクトラム拡散通信、前方誤り訂正、および消失訂正符号を組み合わせた多層的なアーキテクチャを採用しています。

## 概要

本プロジェクトは、マイクとスピーカーを用いたデータ通信（Data-over-Sound）を目的としています。ストリーミング処理（WebAudioなどを想定した細切れのサンプルチャンク処理）において、メモリ破綻や無限ループを起こさず、いかなる境界条件でもパケットをロスしない強固なデコーダパイプラインを備えています。

## ディレクトリ構造とアーキテクチャ

本ライブラリは、通信レイヤーと数学的責務に基づいて以下の通り整理されています。

```text
dsp/src/
├── lib.rs              # 全体設定・パブリックAPI
├── encoder.rs          # 送信オーケストレーション
├── decoder.rs          # 受信オーケストレーション
├── common/             # 数学的プリミティブ (Layer-agnostic)
│   ├── msequence.rs    # M系列生成アルゴリズム
│   ├── crc.rs          # 巡回冗長検査 (CRC-16)
│   ├── rrc_filter.rs   # FIRフィルタ / RRC係数計算
│   ├── nco.rs          # 数値制御発振器 (正弦波生成)
│   └── decimator.rs    # フィルタリングとダウンサンプリング
├── phy/                # 物理層：波形とビットの相互変換
│   ├── modulator.rs    # 変調 (DBPSK/DSSS)
│   ├── demodulator.rs  # 復調
│   └── sync.rs         # 同期捕捉ロジック
├── coding/             # 符号化層：ビット列の信頼性制御
│   ├── fec.rs          # 誤り訂正 (Convolutional/Viterbi)
│   ├── fountain.rs     # 消失訂正 (RLNC over GF(256))
│   └── interleaver.rs  # ブロックインターリーバー
└── frame/              # データリンク層：パケット構造
    └── packet.rs       # パケットフォーマット・シリアライズ
```

## 技術スタックとアーキテクチャ

通信の信頼性を確保するため、以下の技術をパイプラインとして統合しています。

### 1. 物理レイヤー (変調・復調)
- **DSSS (Direct Sequence Spread Spectrum)**: M系列（次数5, 長さ31）を用いた直接拡散。狭帯域ノイズやマルチパス干渉に対して強力な耐性（処理利得）を提供します。
- **DBPSK (Differential Binary Phase Shift Keying)**: 差動位相偏移変調。絶対的な位相基準が不要であり、音響通信における位相の揺らぎや回転に対して強靭です。
- **RRC (Root-Raised Cosine) フィルタ**: ロールオフ係数0.35のパルス整形フィルタ。シンボル間干渉（ISI）を最小化します。
- **キャリア周波数**: デフォルト 8,000 Hz。一般的なオーディオ機器で再生・録音しやすい帯域を使用します。

### 2. 同期捕捉 (Synchronization)
- **スライディング相関とコヒーレント積分**: プリアンブル（M系列の繰り返し）をスライディング相関で探索します。I/Qチャネルを用いた複素コヒーレント積分により、位相回転に依存せず高いS/N比でピークを検出します。
- **厳密なローカルピーク検出**: 誤検出（False Positive）を数学的に排除するため、バッファ内の絶対最大エネルギーを基準とした動的な閾値（90%以上）と局所的極大値の判定を組み合わせています。

### 3. 誤り訂正と消失訂正
- **FEC (Forward Error Correction)**: 畳み込み符号（拘束長 K=7, 符号化率 R=1/2）とViterbi復号による強力なビット単位の誤り訂正。
- **インターリーブ**: バーストエラーを分散させ、Viterbi復号の性能を最大化するブロックインターリーバー。
- **CRC-16**: パケット単位の整合性チェック。
- **RLNC (Random Linear Network Coding)**: GF(256) 上の有限体演算を用いた消失訂正アルゴリズム（Fountain Codeの一種）。送信データは $k$ 個のソースブロックに分割され、各パケットはそれらのランダム（または決定論的な）線形結合として生成されます。受信側で**任意の $k$ 個の線形独立なパケット**が集まれば、ガウス消去法により元のデータを確実に復元可能です。これにより、パケットの到着順序や特定の消失パケットを意識する必要がない強力な Rateless 通信を実現しています。

## パイプライン・データフロー

### 送信 (Encoder)
1. **データ入力** → RLNC符号化により無限のパケットストリームを生成
2. パケットにCRC-16を付加して直列化
3. FEC（畳み込み符号化）とインターリーブ
4. DBPSKによる差動符号化とDSSS（M系列）によるチップ拡散
5. RRCフィルタによる波形整形
6. 8kHzキャリアによるアップコンバート → **音声サンプル (f32) 出力**

### 受信 (Decoder)
1. **音声サンプル入力 (チャンク単位)** → 8kHzキャリアによるダウンコンバート (I/Q)
2. RRCマッチドフィルタとチップレートへのデシメーション
3. 同期捕捉（プリアンブル検出）とバッファ管理
4. M系列による逆拡散とDBPSK差動復号（ビット判定）
5. デインターリーブとViterbi復号による誤り訂正
6. CRC検証を通過したパケットを RLNC デコーダへ投入
7. ガウス消去法による RLNC 復号 → **元データの復元**

## 実行要件・ビルド方法

本ライブラリは Rust で記述されており、標準的な Cargo ワークフローをサポートしています。
将来的には `wasm-bindgen` を通じて Webブラウザ (WebAudio API) から直接利用されることを想定しています。

### テストの実行

すべてのDSPモジュールは、数学的な厳密性を保証するための厳格なテストに保護されています。

```bash
cd dsp
cargo test --release
```

特に `e2e_noise.rs` では、AWGN（加法性ホワイトガウスノイズ）環境下において、ストリームが細切れに入力される過酷な状況でもパケットをロスせず復元できるかを検証しています。

## 設計哲学
- **数学的証明に基づく実装**: 曖昧なアサーションや閾値を排除し、フィルタ遅延や相関ピーク位置の理論値と1サンプルの狂いもなく一致する実装を追求しています。
- **ストリーミングの安全性**: `Decoder` は入力データの長さに関わらず状態を安全に保持し、処理済みバッファの確実なガベージコレクションを行うため、メモリリークや処理の破綻を起こしません。
- **WASM境界の集約**: WASM (JavaScript) に対するインターフェースは `dsp/src/lib.rs` にのみ定義されています。他の内部モジュールは純粋な Rust として保たれ、ポータビリティと単体テストのしやすさを維持しています。
